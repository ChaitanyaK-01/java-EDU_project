Multithreading in Java

What is a Thread?

A thread is the smallest unit of execution within a process. Threads allow concurrent execution of tasks within the same program, enabling better performance and responsiveness.

What is Multithreading?

Multithreading is a programming concept where multiple threads execute independently within the same program, sharing the same resources such as memory space.

How to Declare Threads?

In Java, there are two main ways to create threads:


// Method 1: Extending the Thread class
class MyThread extends Thread {
    public void run() {
        // Thread logic goes here
    }
}

// Method 2: Implementing the Runnable interface
class MyRunnable implements Runnable {
    public void run() {
        // Thread logic goes here
    }
}

public class ThreadDeclarationExample {
    public static void main(String[] args) {
        // Using method 1: Extending Thread class
        MyThread thread1 = new MyThread();
        thread1.start();
        
        // Using method 2: Implementing Runnable interface
        Thread thread2 = new Thread(new MyRunnable());
        thread2.start();
    }
}

Thread vs. Process

The key differences between threads and processes are:

  • Threads share the same memory space, while processes have independent memory spaces.
  • Threads are lightweight compared to processes, making context switching faster.
  • Threads within the same process can communicate directly, while processes typically communicate via inter-process communication mechanisms.

Methods Related to Threads

Java provides several methods to manage threads effectively:

  • sleep(): Suspends the execution of the current thread for a specified period.
  • join(): Waits for a thread to terminate.
  • wait() and notify(): Allows threads to communicate and synchronize their actions.

class ThreadMethodsExample {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            try {
                System.out.println("Thread is running...");
                Thread.sleep(2000); // Sleeping for 2 seconds
                System.out.println("Thread woke up.");
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });

        thread.start();

        try {
            thread.join(); // Wait for the thread to finish
            System.out.println("Thread execution completed.");
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}

Thread Lifecycle

A thread in Java goes through several states during its lifecycle:

  • New: The thread is in a new state before starting.
  • Runnable: The thread is ready to run and waiting for the CPU.
  • Running: The thread is executing its task.
  • Blocked: The thread is blocked waiting for a monitor lock.
  • Terminated: The thread has completed execution.
