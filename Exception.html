Exception Handling in Java

What is an Exception?

An exception is an event, which occurs during the execution of a program, that disrupts the normal flow of the program's instructions. It can occur due to various reasons such as incorrect user input, hardware failure, or programming errors.

What is Exception Handling?

Exception handling is a mechanism to handle runtime errors in a controlled manner. It enables a program to handle exceptional conditions gracefully, preventing abrupt termination.

Checked Exceptions

Checked exceptions are exceptions that are checked at compile-time. They are typically recoverable and should be handled by the programmer.

Types of Checked Exceptions

  • IOException

    IOException is thrown when an I/O operation fails or is interrupted, which can occur when reading from or writing to files, streams, or other sources.

    
import java.io.FileReader;
import java.io.IOException;

public class IOExceptionExample {
    public static void main(String[] args) {
        try {
            FileReader file = new FileReader("example.txt");
            // Perform file operations
        } catch (IOException e) {
            System.out.println("IOException occurred: " + e.getMessage());
            e.printStackTrace();
        }
    }
}

  • SQLException

    SQLException is thrown when there is an error in database access or SQL query execution.

    
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class SQLExceptionExample {
    public static void main(String[] args) {
        try {
            Connection conn = DriverManager.getConnection("jdbc:mysql://localhost:3306/database", "username", "password");
            // Perform database operations
        } catch (SQLException e) {
            System.out.println("SQLException occurred: " + e.getMessage());
            e.printStackTrace();
        }
    }
}

  • ClassNotFoundException

    ClassNotFoundException is thrown when an application tries to load a class through its string name using methods like `Class.forName()`, but the specified class cannot be found in the classpath.

    
public class ClassNotFoundExceptionExample {
    public static void main(String[] args) {
        try {
            Class.forName("com.example.NonExistingClass");
        } catch (ClassNotFoundException e) {
            System.out.println("ClassNotFoundException occurred: " + e.getMessage());
            e.printStackTrace();
        }
    }
}

Unchecked Exceptions (RuntimeExceptions)

Unchecked exceptions, also known as runtime exceptions, are not checked at compile-time. They typically occur due to programming bugs and are not expected to be handled.

Types of Unchecked Exceptions

  • NullPointerException

    NullPointerException is thrown when a program attempts to use an object reference that has not been initialized (null).

    
public class NullPointerExceptionExample {
    public static void main(String[] args) {
        String str = null;
        try {
            int length = str.length(); // This will throw NullPointerException
        } catch (NullPointerException e) {
            System.out.println("NullPointerException occurred!");
            e.printStackTrace();
        }
    }
}

  • ArrayIndexOutOfBoundsException

    ArrayIndexOutOfBoundsException is thrown when attempting to access an array element at an invalid index.

    
public class ArrayIndexOutOfBoundsExceptionExample {
    public static void main(String[] args) {
        int[] arr = {1, 2, 3};
        try {
            int value = arr[5]; // This will throw ArrayIndexOutOfBoundsException
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("ArrayIndexOutOfBoundsException occurred!");
            e.printStackTrace();
        }
    }
}

  • ArithmeticException

    ArithmeticException is thrown when an exceptional arithmetic condition occurs, such as divide by zero.

    
public class ArithmeticExceptionExample {
    public static void main(String[] args) {
        try {
            int result = 10 / 0; // This will throw ArithmeticException
        } catch (ArithmeticException e) {
            System.out.println("ArithmeticException occurred!");
            e.printStackTrace();
        }
    }
}

  • NumberFormatException

    NumberFormatException is thrown when an attempt is made to convert a string into a numeric format, but the string does not have the appropriate format.

    
public class NumberFormatExceptionExample {
    public static void main(String[] args) {
        String str = "abc";
        try {
            int num = Integer.parseInt(str); // This will throw NumberFormatException
        } catch (NumberFormatException e) {
            System.out.println("NumberFormatException occurred: " + e.getMessage());
            e.printStackTrace();
        }
    }
}
